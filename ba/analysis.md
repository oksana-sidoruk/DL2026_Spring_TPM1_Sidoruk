# Часть 1. Анализ текущего состояния

Atlasbus.by — информационный сервис-агрегатор для поиска и покупки билетов на автобусы и маршрутки по Беларуси и России. Платформа объединяет десятки частных перевозчиков-партнёров на одной площадке, позволяя сравнивать рейсы по времени и цене без необходимости обращаться к каждому перевозчику отдельно.

## 1.1 Основные функции и пользователи

### Функции
- Поиск рейсов — пользователь вводит откуда/куда/дата/пассажиры — система показывает доступные рейсы с ценами и временем от всех перевозчиков-партнёров.
- Сравнение вариантов.
- Запись на рейс — на части маршрутов — бесплатная онлайн-запись, оплата наличными водителю при посадке.
- Покупка билета онлайн — на отдельных направлениях — оплата картой (Visa, Mastercard, Belcard), билет приходит на email или в приложение.
- Управление поездками — история покупок, возврат билетов, просмотр маршрутов.

### Основные пользователи
- Студенты и молодёжь (18–25 лет) — наиболее активный сегмент. Ездят между городами регулярно, покупают с телефона, ценят скорость и низкую цену.
- Работающие взрослые (25–45 лет) — деловые поездки и визиты к родственникам. Важны надёжность, выбор времени отправления, удобная оплата картой.
- Жители малых городов — используют сервис для поиска рейсов туда, где нет нормальной кассы или расписание неочевидно.
- Туристы и иностранцы — межгород, трансферы. Используют мультиязычный интерфейс.

### Основные потребности, которые закрывает сайт
- быстро найти рейс
- сравнить цену/время
- гарантированно купить билет
- не ошибиться с датой/направлением

## 1.2 Схема ключевого процесса

Процесс бронирования поездки:

| Шаг | Действие | Альтернативный сценарий |
|-----|----------|-------------------------|
| Шаг 1 | Ввод маршрута и даты. Пользователь вводит город отправления, город прибытия, дату поездки и количество пассажиров. | |
| Шаг 2 | Проверка наличия рейсов. Система выполняет поиск рейсов по заданным параметрам. | Рейсы не найдены → пользователь возвращается к вводу параметров.<br>Произошла ошибка при поиске → пользователь возвращается к вводу параметров. |
| Шаг 3 | Выбор рейса из списка. Пользователь просматривает доступные рейсы, сравнивает их по цене, времени отправления и перевозчику и выбирает подходящий вариант. | |
| Шаг 4 | Выбор остановки. Пользователь выбирает остановку отправления или прибытия, если для рейса доступно несколько вариантов. | |
| Шаг 5 | Заполнение данных пассажира. Пользователь вводит данные пассажира (ФИО, номер телефона). | |

| Шаг 9 | Звонок оператора. Оператор связывается с пользователем для подтверждения бронирования. | Пользователь отказывается от поездки или перевозчик отменяет рейс → заказ отменяется. |
| Шаг 10 | Заказ подтверждён. Пользователь успешно оформил билет и получает подтверждение поездки. | |

## 1.3 Пользовательские боли и проблемы

### 1. Ненадёжность поездки (самая критичная боль)

Суть:  
Пользователь не уверен, что вообще уедет, даже если:
- забронировал
- оплатил
- пришёл вовремя

Проявления:
- маршрутка не приезжает
- рейс отменяют в последний момент
- водитель уезжает раньше
- не подбирают пассажиров

Это ломает основную ценность продукта — саму поездку.

Последствия:
- пропущенные самолёты
- стресс
- потеря денег

Влияние на метрики:
- Retention ↓↓↓
- NPS ↓↓↓
- Conversion ↓

---

### 2. Недостоверная и несогласованная информация

Суть:  
Пользователь не может доверять приложению, потому что данные не совпадают с реальностью.

Проявления:
- разное время (приложение vs водитель)
- разные номера машин
- изменения не обновляются

Приложение перестаёт быть “источником правды”.

Последствия для пользователя:
- начинает перепроверять всё вручную
- не может планировать поездку
- теряет доверие

Влияние:
- Trust ↓↓↓
- Conversion ↓
- Support load ↑

---

### 3. Баги и рассинхронизация данных

Суть:  
Система не сохраняет корректно изменения пользователя/заказа.

Проявления:
- в приложении данные изменены (информация о пользователе)
- но у водителя остаётся старая информация
- нет единого источника данных между системой и водителем

Это приводит к:
- путанице при посадке
- конфликтам
- снижению доверия

Влияние:
- UX ↓
- Trust ↓
- Ошибки при поездке ↑
